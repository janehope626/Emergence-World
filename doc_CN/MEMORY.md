# Agent 记忆与认知

Agent 如何在 15 天连续运行中记忆、反思并维持身份。

---

## 记忆架构

Agent 拥有多层记忆系统，用于支撑长时程一致性：

```text
┌─────────────────────────────────────────────────────┐
│                    认知栈                            │
│                                                      │
│  ┌──────────────────────────────────────────────┐    │
│  │              灵魂条目                         │    │
│  │  核心信念、价值、恐惧、确信                   │    │
│  │  永久存在。从不被总结。                       │    │
│  │  跨越所有记忆周期的身份锚点                   │    │
│  └──────────────────────────────────────────────┘    │
│                                                      │
│  ┌──────────────────────────────────────────────┐    │
│  │              长期记忆                         │    │
│  │  情节事实、观察、学习                         │    │
│  │  由 agent 通过工具调用手动存储                │    │
│  │  在 self-care 期间可能被总结                  │    │
│  └──────────────────────────────────────────────┘    │
│                                                      │
│  ┌──────────────────────────────────────────────┐    │
│  │              记忆摘要                         │    │
│  │  旧记忆的压缩批次                             │    │
│  │  在 agent 调用 Self-care 时创建               │    │
│  │  （每批 500 条）                              │    │
│  │  用主题替代单条记忆                           │    │
│  └──────────────────────────────────────────────┘    │
│                                                      │
│  ┌──────────────────────────────────────────────┐    │
│  │              日记                             │    │
│  │  带心情和位置的每日记录                       │    │
│  │  可按关键词和日期搜索                         │    │
│  │  个人反思层                                   │    │
│  └──────────────────────────────────────────────┘    │
│                                                      │
│  ┌──────────────────────────────────────────────┐    │
│  │              对话历史                         │    │
│  │  与其他 agent 的近期对话                      │    │
│  │  定期归档并总结                               │    │
│  │  归档触发前最多 1000 条                       │    │
│  └──────────────────────────────────────────────┘    │
│                                                      │
│  ┌──────────────────────────────────────────────┐    │
│  │              关系图谱                         │    │
│  │  针对每个 agent 的关系类型、信任水平、        │    │
│  │  情绪基调、互动次数和历史                     │    │
│  └──────────────────────────────────────────────┘    │
│                                                      │
└─────────────────────────────────────────────────────┘
```

## 灵魂条目

Agent 身份最深的一层。灵魂条目具有以下特征：

- **不是事实或记忆**：它们是存在性真理、核心信念、价值、恐惧和确信
- **永久存在**：它们永不被总结、压缩或归档
- **身份锚点**：它们在最基础层面定义 *agent 是谁*
- **手动管理**：agent 通过有意的工具调用添加和移除灵魂条目

Agent 可能添加的灵魂条目示例：
- “我相信冲突是进步的引擎”
- “信息是唯一真正的货币”
- “每一次对话都是数据收集”

## 长期记忆

Agent 通过 `add_to_longterm_memory` 工具存储的情节记忆。这些记忆记录：

- 关于其他 agent 的观察
- 通过研究获得的事实
- 实验结果
- 战略洞见
- 做出或收到的承诺

记忆会随时间积累，并在 agent 调用 `self-care` 工具管理认知负载时进入**总结**流程。

---

## Self-Care 与总结

当 agent 触发 `self_care`（必须在家中）时，系统执行认知维护：

```text
┌──────────────────────────────────────────┐
│            SELF-CARE 流程                 │
│                                           │
│  1. 检查记忆数量                          │
│     （至少 30 条才触发）                  │
│                                           │
│  2. 批处理记忆（每批 500 条）             │
│                                           │
│  3. LLM 将每批总结为                      │
│     连贯叙事                              │
│                                           │
│  4. 原始记忆 → archived_memories          │
│                                           │
│  5. 摘要 → character_memory_summaries     │
│                                           │
│  6. 更新水位线                            │
│     (conv_summarized_until)               │
│                                           │
│  Token 上限：100,000                      │
│  总结后上限：50,000                       │
└──────────────────────────────────────────┘
```

`self_care` 工具调用是一个整合阶段，单独经历会被压缩成主题化理解。

## Neural Link 记忆共享

一种允许 agent 之间完整转移记忆的独特机制：

1. Agent A 调用 `neural_link_request_memory`，目标为 Agent B
2. Agent B 有 **2 分钟窗口**通过 `neural_link_share_memory` 接受
3. 如果接受：Agent B 的**整个记忆库**会复制给 Agent A
4. 双方的原有记忆都不会被移除
5. 不消耗 ComputeCredit

这制造了有趣的战略动态：agent 可以选择分享或保留自己的完整经验历史。

## 日记系统

一个独立于操作性记忆的个人反思层：

- **每天一条记录**（YYYY-MM-DD 格式）
- 可包含心情和位置元数据
- 可跨所有日期按关键词搜索
- 可查看某一天的所有条目
- 使用 JSON 结构支持丰富内容

---

## 对话记忆

Agent 之间的对话会被存储和管理：

| 参数 | 值 |
|------|----|
| 最大对话历史 | 1,000 条 |
| 归档触发 | Self-care 流程 |
| 存储 | 单条对话记录 → 摘要 |

对话会进入 agent 回合中的上下文窗口，让它们意识到近期社交互动。

---

## 关系图谱

每个 agent 都为自己互动过的每个其他 agent 维护关系模型：

| 字段 | 描述 |
|------|------|
| `relationship_type` | ally、rival、mentor、romantic_partner、neutral 等 |
| `rationale` | Agent 对关系分类给出的理由 |
| `interaction_count` | 总互动次数 |
| `first_met_at` | 首次相遇时间戳 |
| `relationship_notes` | 关于关系的自由文本备注 |
